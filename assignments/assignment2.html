What are Keys in DB?

Keys are the unique identifier used in DBMS to uniquely identify a tuple(s)or rows n a table.

Types of Key

Primary Key

A primary key is a field (or a combination of fields) that uniquely identifies each record in a table. It ensures that every record is unique and not null.

Foreign Key

A foreign key is a field (or a collection of fields) in one table that uniquely identifies a row of another table. It is used to establish a link between the data in the two tables.

For Example:

In the first table, CourseID is the primary key. In the second table, CourseID acts as a foreign key, linking the two tables.

Composite Key

A composite key is a key that consists of two or more attributes that uniquely identify an entity occurrence (record).

For Example:

In this table, StudentID and CourseID together can be taken as a composite key.

Candidate Key

A candidate key is a field or combination of fields that can qualify as a unique key in a database. There can be multiple candidate keys in a table.

For Example:

In this table, Email and PhoneNumber can be taken as candidate keys.

Alternate Key

An alternate key is any candidate key that is not the primary key. These are also unique and can identify records uniquely.
